Présentation

  • Framework crée par Pivotal
  • Première GA en Avril 2014
  • 18 mois de développement

Concept

  • Simplifier la création d'application Spring
  • Améliorer la productivité
  • Reduire la configuration
  • Simplifier le déploiement
  • Propsoser l'opinon Spring

Fonctionnalités

  • Packaging en JAR ou WAR
  • Des pom.xml "starters" permettant d'activer facilement des modules (db, mail, jpa, ...)
  • Mécanisme d'autoconfiguration
  • Gestion des profiles et des properties
  • Outillage pour la production (metriques et monitoring)

Modules

  • logging (slf4j, logback, log4j2)
  • data (jpa, es, mongo, solr, gemfire, redis)
  • messaging (amqp, hornetq)
  • templating (freemarker, thymeleaf, velocity)
  • container (jetty, tomcat, undertow)
  • web (spring-mvc, websocket, jersey, ws)
  • mail
  • ...

Démarrage

  • CLI
    • Ligne de commande, prototypage en groovy
  • Maven, Gradle
    • POM parent , Plugin
  • http://start.spring.io
    • Initialise un projet Spring Boot
    • Choix des modules, de l'outil de build, version

Démo

Comment ça marche ?

  • Heritage de POMs
  • Factories Spring
  • Annotations

Gestion des POMs

  • Projet herite de "spring-boot-parent"
  • "spring-boot-dependencies" spécifie versions JARs
  • "spring-boot-starter-XXX" ajoute les dépendances

Spring Factories

  • Spring lit les fichiers META-INF/spring.factories
  • Instanciation une classe d'autoconfiguration

Annotations

  • @EnableAutoconfiguration(exclude=???)
  • @Conditionnal
  • @EnableWebMvc, @EnableTransactionManagement
